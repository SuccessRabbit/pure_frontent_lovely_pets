# 资源素材系统说明 - 更新版

## 🎨 新的资源结构

### 卡牌组成方式

卡牌不再是整张SVG，而是通过**代码动态组装**多个元素：

```
┌─────────────────────┐
│  [费用]      [稀有度] │  ← 右上角费用圆圈 + 左上角稀有度标签
├─────────────────────┤
│                     │
│   角色插画（4:3）    │  ← 独立的插画资源
│                     │
├─────────────────────┤
│    【卡牌名称】      │  ← 文字渲染
│    ─────────        │  ← 分隔线
│   ⭐2  💢1/3        │  ← 属性图标（动态生成）
│                     │
│   效果描述文字...    │  ← 从配置读取
│   #标签 #标签       │  ← 标签列表
└─────────────────────┘
```

## 🎯 卡牌组件使用

### 基础用法

```tsx
import { Card } from '@/components/Card';
import petCard from '@config/pets.json';

// 卡牌会自动组装所有元素
<Card card={petCard[0]} onClick={() => console.log('clicked')} />
```

### 卡牌自动包含

1. **费用标识**：右上角圆形，显示小罐头消耗
2. **稀有度标签**：左上角，根据稀有度显示不同颜色
3. **角色插画**：4:3比例，自动PNG→SVG fallback
4. **卡牌名称**：居中显示
5. **属性图标**：收益⭐、压力💢（实体卡专属）
6. **效果描述**：从配置文件读取
7. **标签**：显示前3个标签
8. **悬停效果**：缩放、光效、阴影

### 稀有度配色

```typescript
普通（common）：灰色边框 + 灰色渐变
稀有（rare）：蓝色边框 + 蓝色渐变
史诗（epic）：紫色边框 + 紫色渐变
传说（legendary）：金色边框 + 金色渐变
```

## 📐 插画规范

### 尺寸要求
- **SVG viewBox**：`0 0 400 300`（4:3比例）
- **PNG尺寸**：400×300px 或更高（保持4:3）

### 设计要求
- **主体居中**：角色应在画面中央
- **留白充足**：四周留10%边距，避免被裁切
- **背景简洁**：纯色或简单渐变
- **风格统一**：温暖萌宠风格，圆润可爱

### 色彩建议
- **萌宠**：暖色调（橙、粉、黄）
- **牛马**：冷色调（灰、蓝）
- **指令**：鲜艳色（根据功能）

## 🔧 资源加载机制

### 智能Fallback

```typescript
// 1. 尝试加载PNG
/assets/illustrations/pets/pet_001.png

// 2. PNG不存在，自动使用SVG
/assets/illustrations/pets/pet_001.svg

// 3. 无缝显示，用户无感知
```

### 添加自定义PNG

只需将PNG文件放入对应目录：

```bash
# 萌宠插画
public/assets/illustrations/pets/pet_001.png

# 牛马插画
public/assets/illustrations/workers/worker_001.png

# 指令图标
public/assets/illustrations/actions/action_001.png
```

刷新浏览器，系统自动优先加载PNG。

## 🎨 与旧版本的区别

### 旧版本（已废弃）
```
/assets/cards/pets/pet_001.svg  ← 整张卡牌的SVG
```
- 包含卡牌框架、文字、图标
- 难以修改和维护
- 无法动态调整

### 新版本（当前）
```
/assets/illustrations/pets/pet_001.svg  ← 只有角色插画
```
- 只包含角色本身
- 卡牌元素由代码组装
- 灵活可配置
- 易于替换和更新
