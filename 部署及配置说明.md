# 萌宠直播公司部署及配置说明

## 1. 文档范围

本仓库只保留两份正式文档：

- `部署及配置说明.md`：开发、构建、部署、配置编辑与资源约定
- `项目技术文档.md`：运行时架构、模块职责、规则与数据流

本文档以当前代码实现为准。

## 2. 环境要求

- Node.js `>= 18`
- npm `>= 9`

## 3. 项目启动

安装依赖：

```bash
npm install
```

启动本地开发环境：

```bash
npm run dev
```

该命令会同时启动 3 个进程：

- `scripts/compile-data.js`：把 `data/*.csv` 编译为 `config/*`
- `scripts/admin-api.js`：启动本地 Admin API，默认 `http://127.0.0.1:3001`
- `npm run dev:web`：启动 Vite，默认 `http://localhost:3000`

本地主要入口：

- 游戏：`http://localhost:3000`
- 配置后台：`http://localhost:3000/admin`
- 本地配置接口健康检查：`http://127.0.0.1:3001/health`

## 4. 构建与预览

生产构建：

```bash
npm run build
```

构建流程：

1. 运行 `npm run compile:data`
2. 执行 `tsc`
3. 执行 `vite build`
4. 输出到 `dist/`

本地预览：

```bash
npm run preview
```

辅助命令：

```bash
npm run type-check
npm run lint
```

## 5. 部署方式

### 5.1 Vercel

仓库已提供 `vercel.json`：

- 构建命令：`npm run build`
- 产物目录：`dist`
- SPA 路由重写到 `index.html`
- `/assets/*` 设置长期缓存

直接接入 Vercel 时，无需额外后端服务。线上环境不会启用本地 Admin API，`/admin` 会退化为只读配置查看页。

### 5.2 静态托管

只要支持单页应用回退到 `index.html`，也可以部署到其他静态托管平台。关键要求：

- 先执行 `npm run build`
- 发布 `dist/`
- 配置 SPA rewrite

## 6. 配置源与生成产物

### 6.1 唯一配置源

当前唯一配置源为 `data/*.csv`：

- `data/cards.csv`：卡牌基础字段、资源路径、模型绑定
- `data/card_skills.csv`：卡牌与技能模板的绑定关系
- `data/skill_templates.csv`：技能模板与参数 schema
- `data/model_profiles.csv`：宠物 3D 模型预设
- `data/global_config.csv`：全局规则参数

### 6.2 编译产物

`npm run compile:data` 会生成：

- `config/pets.json`
- `config/workers.json`
- `config/actions.json`
- `config/cards.json`
- `config/runtimeConfig.json`
- `config/gameRules.ts`
- `src/types/cards.generated.ts`

这些文件应视为编译产物，正常维护时优先编辑 `data/*.csv`，不要把手工改动长期停留在生成文件里。

## 7. Admin 编辑工作流

推荐本地配置流程：

1. 执行 `npm run dev`
2. 打开 `http://localhost:3000/admin`
3. 编辑卡牌、技能模板、技能绑定、模型预设和全局参数
4. 点击保存
5. Admin API 回写 CSV，并重新编译 `config/*`
6. 游戏页收到事件后自动刷新

注意事项：

- 只有在 `localhost` 或 `127.0.0.1` 下，前端才会尝试连接 `http://127.0.0.1:3001`
- 非本地环境的 `/admin` 为只读模式
- 保存失败时，服务端会回滚本次 CSV 写入

## 8. 资源目录约定

静态资源位于 `public/assets/`：

- `public/assets/cards/`：当前配置默认引用的卡面/贴图目录
- `public/assets/illustrations/`：独立插画目录，运行时可作为回退来源
- `public/assets/ui/icons/`：HUD 与后台使用的图标资源

命名规则：

- 宠物：`pet_001`
- 牛马：`worker_001`
- 行动牌：`action_001`
- 文件名应与卡牌 `id` 一致

运行时会优先尝试配置中给出的路径，并按顺序回退到同名 PNG/SVG 与兼容目录，因此新增素材时应尽量保持同名同分类。

## 9. 常见修改入口

数值、描述、卡牌技能：

- 优先编辑 `data/*.csv`
- 或通过 `/admin` 编辑并保存

全局规则参数：

- 编辑 `data/global_config.csv`

模型预设：

- 编辑 `data/model_profiles.csv`

资源替换：

- 替换 `public/assets/` 下对应同名文件

## 10. 发布前检查

发布前至少执行：

```bash
npm run compile:data
npm run type-check
npm run build
```

建议人工确认：

- `http://localhost:3000` 能正常进入游戏
- `http://localhost:3000/admin` 能读取配置
- 卡牌图片、3D 宠物和 HUD 图标显示正常
- 修改配置后能生成新的 `config/runtimeConfig.json`
