# 萌宠直播公司 - 技术选型文档

## 一、技术栈选型

### 1.1 前端框架
**选择：React 18 + TypeScript**

**理由：**
- React 生态成熟，组件化开发效率高
- TypeScript 提供类型安全，减少卡牌配置错误
- Hooks 机制适合游戏状态管理
- Vercel 对 React 项目支持最佳
- 社区资源丰富，易于后期扩展

### 1.2 游戏渲染引擎
**选择：React + CSS Animations + Canvas（混合方案）**

**理由：**
- UI 层（卡牌、网格、面板）使用 React + CSS，开发效率高
- 特效层（粒子、拆家动画）使用 Canvas，性能更好
- 不使用重量级游戏引擎（Phaser/PixiJS），减小包体积
- 纯前端实现，无需额外学习成本

**技术细节：**
```
UI 层（React）
├── 网格系统（Grid Layout）
├── 卡牌展示（Flexbox + CSS Transform）
├── 手牌区域（Drag & Drop API）
└── 资源面板（状态显示）

特效层（Canvas）
├── 粒子系统（拆家碎片）
├── 连线效果（羁绊提示）
└── 压力条动画（渐变填充）
```

### 1.3 状态管理
**选择：Zustand**

**理由：**
- 轻量级（~1KB），适合游戏快速迭代
- API 简洁，无需 Provider 包裹
- 支持中间件（持久化、时间旅行调试）
- 性能优于 Redux，适合高频状态更新

**状态结构：**
```typescript
interface GameState {
  // 网格状态
  grid: GridCell[][];

  // 卡牌状态
  deck: Card[];
  hand: Card[];
  discard: Card[];

  // 资源状态
  cans: number;        // 小罐头
  hearts: number;      // 小红心（分数）
  day: number;         // 当前天数

  // 经济状态
  interest: number;    // 利息
  winStreak: number;   // 连胜
  loseStreak: number;  // 连败
}
```

### 1.4 构建工具
**选择：Vite**

**理由：**
- 开发服务器启动速度极快（<1s）
- HMR 热更新体验好，适合调试游戏数值
- 打包体积小，Tree-shaking 效果好
- 原生支持 TypeScript 和 JSON 导入
- Vercel 完美支持

### 1.5 样式方案
**选择：Tailwind CSS + CSS Modules**

**理由：**
- Tailwind 快速搭建 UI，原子化 class 易于维护
- CSS Modules 处理复杂动画和特殊效果
- 支持主题切换（未来可扩展皮肤系统）

## 二、配置文件系统

### 2.1 卡牌配置格式
**选择：JSON（运行时） + Excel（编辑时）**

**文件结构：**
```
/config
├── cards.json          # 卡牌配置（自动生成）
├── pets.json           # 萌宠配置
├── workers.json        # 牛马配置
├── items.json          # 道具配置
└── synergies.json      # 羁绊配置

/excel
├── cards.xlsx          # Excel 编辑源文件
└── converter.js        # 转换脚本
```

**Excel 表格结构示例：**
| ID | 名称 | 类型 | 费用 | 描述 | 收益 | 压力 | 特殊效果 | 稀有度 |
|----|------|------|------|------|------|------|----------|--------|
| pet_001 | 新晋柯基 | 实体-萌宠 | 2 | 可爱但容易焦虑 | 2 | 1 | 相邻牛马效率+10% | 普通 |
| worker_001 | 实习牛马 | 实体-牛马 | 1 | 廉价劳动力 | 1 | 0 | 可献祭 | 普通 |
| action_001 | 高级猫薄荷 | 指令 | 1 | 清零压力 | 0 | -3 | 目标萌宠压力归零 | 稀有 |

### 2.2 Excel 转换脚本
**技术选型：Node.js + xlsx 库**

```bash
# 安装依赖
npm install xlsx

# 转换命令
npm run convert:cards
```

**脚本功能：**
- 读取 Excel 文件
- 数据验证（必填项、数值范围）
- 生成 TypeScript 类型定义
- 输出 JSON 配置文件
- 生成卡牌 ID 索引

## 三、资源管理方案

### 3.1 图像资源结构
```
/public/assets
├── cards/              # 卡牌图片
│   ├── pets/          # 萌宠卡面
│   ├── workers/       # 牛马卡面
│   ├── actions/       # 指令卡面
│   └── frames/        # 卡牌边框
├── ui/                # UI 元素
│   ├── icons/         # 图标（小罐头、小红心）
│   ├── buttons/       # 按钮
│   └── backgrounds/   # 背景
├── effects/           # 特效素材
│   ├── particles/     # 粒子贴图
│   └── animations/    # 动画序列帧
└── grid/              # 网格元素
    ├── cells/         # 工位背景
    └── decorations/   # 装饰物
```

### 3.2 资源加载策略
**选择：懒加载 + 预加载关键资源**

```typescript
// 关键资源（游戏启动时加载）
const CRITICAL_ASSETS = [
  '/assets/ui/icons/can.png',
  '/assets/ui/icons/heart.png',
  '/assets/cards/frames/common.png'
];

// 按需加载（卡牌首次出现时加载）
const loadCardImage = (cardId: string) => {
  return import(`/assets/cards/${cardId}.png`);
};
```

### 3.3 占位符方案
**在图片未准备好时使用：**
- 纯色背景 + 文字描述
- SVG 图标（可内联，无需加载）
- CSS 渐变生成的临时卡面

## 六、技术风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| 卡牌数量过多导致配置文件臃肿 | 加载慢 | 按稀有度分文件，懒加载 |
| Canvas 动画性能问题 | 卡顿 | 使用 OffscreenCanvas + Web Worker |
| 状态更新频繁导致重渲染 | 掉帧 | React.memo + useMemo 优化 |
| Excel 转换脚本出错 | 配置错误 | 添加 JSON Schema 验证 |
