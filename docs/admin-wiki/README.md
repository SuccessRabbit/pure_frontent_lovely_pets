# Admin 前端 Wiki

## 适用范围

本文档面向使用 `/admin` 的配置维护人员，解释后台页面结构、保存机制和角色分工。后台仍以 `data/*.csv` 为唯一配置源，前端提供的是更安全的可视化编辑入口。

## 页面地图

- 卡牌与技能：维护卡牌基础字段、数值、资源、3D 模型预设绑定，以及卡牌与技能模板的绑定参数。
- 模板工作台：维护 `skillTemplates`，包括触发时机、目标模式、参数 schema、operations 执行链和模板文案。
- 全局参数：维护通用规则参数和起始牌堆。

## 保存机制

本地开发环境下，访问 `http://localhost:3000/admin` 会连接 `http://127.0.0.1:3001`。点击保存后，前端会提交完整配置草稿，Admin API 会回写 CSV 并重新编译 `config/*`。

线上或未启动 Admin API 时，后台会进入只读模式。只读模式可以查看配置，但不能写回 CSV。

## 角色入口

- 运营人员优先阅读 [operator-guide.md](./operator-guide.md)，重点关注如何新建卡牌、编辑文案数值、绑定技能和保存验证。
- 策划/系统设计人员优先阅读 [designer-guide.md](./designer-guide.md)，重点关注模板机制、参数 schema、触发器、目标模式和 operations。

## 基本流程

1. 启动 `npm run dev`。
2. 打开 `http://localhost:3000/admin`。
3. 在对应功能页修改配置。
4. 修复顶部提示的阻断问题。
5. 点击保存。
6. 回到游戏页确认配置表现。
